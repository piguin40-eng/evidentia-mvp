# Evidentia - cierre para piloto vendible

Fecha: 2026-06-17

## Veredicto

Evidentia no debe presentarse como producto clinico final. El posicionamiento correcto es vender un piloto local-first de memoria vectorial privada: una forma de empezar a guardar el conocimiento y contexto de una persona, equipo o centro para construir una base consultable propia.

La frase comercial segura:

> Evidentia es un nodo local para convertir conocimiento, archivos, decisiones, conversaciones, fotos, videos, PDF, audios transcritos y notas en una base vectorial propia, consultable con fuentes.

Dental puede ser el primer vertical, pero no la esencia del producto.

## Alcance congelado para vender piloto

Incluido:

- App local en Mac/ordenador del cliente.
- SQLite local para registros estructurados.
- Chroma local para RAG.
- Upload real de archivos.
- Analisis visual local basico de imagen/video.
- Chat con el mapa de conocimiento.
- Busqueda local.
- Documento de permisos/consentimiento para datos sensibles.
- Pack/export operativo.
- Panel de inteligencia operativa: ROI inicial, busquedas evitadas, casos similares, memoria de expertos, contradicciones y riesgo.
- Separacion conceptual entre identidad/datos sensibles y conocimiento reutilizable.
- Instalacion demostrable en local.

No incluido todavia:

- Diagnostico clinico validado.
- OCR robusto.
- Transcripcion automatica fiable dentro de la app.
- Multiusuario real con permisos.
- Login/organizaciones.
- Cloud productivo.
- Facturacion/licencias.
- Auditoria legal completa RGPD.
- Vision semantica especializada validada.
- Garantias medicas o regulatorias.

## Que hay que terminar antes de ensenarlo

Prioridad P0, imprescindible:

1. Preparar demo limpia con datos ficticios.
2. Probar flujo completo: crear registro -> subir TXT/PDF/imagen/video -> guardar -> chat -> busqueda -> permisos -> pack.
3. Tener frase de limites visible y verbal: no diagnostico, no datos identificables/sensibles sin permiso.
4. Tener guia de demo de 10 minutos.
5. Tener backup de la carpeta antes de la demo.
6. Verificar URL local y, si hace falta remoto, tunnel temporal estable.

Prioridad P1, para vender mejor:

1. Mejorar web premium con Faki.
2. Meter 3 registros demo de alto impacto.
3. Crear pantalla de estado piloto con que funciona y que no.
4. Preparar una hoja de precio piloto.
5. Grabar video corto de uso.

## Oferta piloto recomendada

Nombre: Piloto Evidentia Local Node.

Duracion: 30 dias.

Cliente ideal: profesional, equipo, centro, clinica, laboratorio o empresa de confianza que acepte probar una herramienta en fase piloto.

Promesa:

- Ordenar conocimiento real del cliente.
- Recuperar contexto, decisiones y aprendizajes pasados.
- Reducir perdida de informacion entre fotos, audios, PDF, notas, conversaciones y memoria humana.
- Crear una base vectorial propia dentro del entorno del cliente.

Condicion critica:

- No se vende como diagnostico.
- Se vende como memoria/conocimiento operativo con supervision humana.

## Precio piloto posible

Opcion conservadora:
- Setup piloto: 300-600 EUR.
- Soporte/test 30 dias: incluido.
- Si continua: mensualidad a definir.

Opcion ambiciosa:
- Setup piloto premium: 900-1500 EUR.
- Incluye instalacion, carga inicial, formacion y configuracion de permisos.

Recomendacion:
Empezar con 600-900 EUR si el comprador es de confianza y entiende que participa en piloto. Gratis solo si aporta feedback serio, datos anonimizados y test semanal.

## Demo de 10 minutos

1. Abrir app.
2. Explicar en 20 segundos:
   "Esto convierte el conocimiento que ahora vive disperso en fotos, videos, audios, PDFs, chats y notas en un mapa consultable propio del cliente."
3. Crear registro demo sin datos reales.
4. Pegar una transcripcion corta.
5. Subir una foto, un PDF/TXT y un video corto.
6. Guardar en el mapa.
7. Entrar en Chat y preguntar:
   "Que sabemos sobre este tema?"
   "Que registros parecidos tengo?"
   "Que problemas o aprendizajes se registraron?"
8. Abrir **Intel** y mostrar ROI, riesgo, memorias de expertos, contradicciones y casos conectados.
9. Mostrar fuentes/registros.
10. Mostrar permisos.
11. Cerrar con:
   "No sustituye criterio humano. Evita que el conocimiento del equipo se pierda."

## Criterio para decir que esta terminado para piloto

- El flujo demo no falla.
- La app no promete cosas que no hace.
- El comprador entiende el valor en menos de 5 minutos.
- Puede meter un registro anonimo y recuperarlo.
- Hay permisos y limites claros.
- Miguel puede explicar el precio piloto sin disculparse.
