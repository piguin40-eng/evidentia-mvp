# Evidentia Sellable Loop

Fuente: orden operativa de Miguel, 2026-06-22. Reforzada por Miguel el 2026-06-24: el cron no se estaba cumpliendo con suficiente fuerza y hay que trabajar todas las mananas para hacer Evidentia vendible antes de vacaciones de agosto.

## Regla Principal

Evidentia no se considera terminada hasta que app y web esten vendibles. El equipo debe trabajar en ciclo cerrado hasta que todo este en estado OK para vender, demostrar y pilotar.

## Objetivo

Convertir Evidentia en una app/web vendible para pilotos fundadores y SEPES:

- Landing clara, creible y orientada a venta.
- App local usable para demo.
- Flujo de ingestion y consulta comprensible.
- Evidencia visual suficiente para explicar el producto.
- Riesgos legales/comerciales bien acotados.
- Demo reproducible sin depender de promesas.
- Fecha limite interna: 31 de julio de 2026. Antes de que Miguel se vaya de vacaciones en agosto debe existir un paquete vendible, demostrable y defendible.

## Ciclo Diario

Hora obligatoria: todas las mananas antes de las 09:00 Europe/Madrid.

1. Revisar estado real de app, web, demo, venta y operacion.
2. Detectar el bloqueo numero 1 que impide vender.
3. Elegir maximo 3 acciones de mayor impacto.
4. Ejecutar al menos una mejora material en archivos, demo, QA, guion, video, instalacion o venta. No basta con analizar.
5. Verificar con una prueba minima: build, endpoint, screenshot, flujo manual, copy review, video, checklist o diff.
6. Actualizar score de vendibilidad y SELLABLE_STATUS.md.
7. Reportar a Miguel solo: que se hizo, que se verifico, que falta, y proxima accion.

## Score De Vendibilidad

Cada vuelta debe puntuar de 0 a 10:

- Web: propuesta, copy, visual, mobile, CTA, confianza.
- App: flujo, estabilidad, demo, datos, feedback de errores.
- Producto: tesis, caso de uso, diferencial, limites.
- Venta: oferta, piloto, one-pager, objeciones, proxima accion.
- Operacion: instalacion, seguridad, backup, reproducibilidad.

## Condicion De Salida

El loop solo puede relajarse cuando:

- Web >= 8/10.
- App demo >= 8/10.
- Piloto fundador definido.
- Demo reproducible.
- Riesgos legales/comerciales escritos.
- Miguel lo valida como vendible.

## Regla Dura

No maquillar el estado. Si algo no esta vendible, decirlo y trabajar el siguiente bloqueo. Mejor una correccion real que un reporte bonito.

## Regla Anti-Informe

Cada ejecucion debe dejar una huella verificable. Valen: archivo editado, test pasado, captura QA, video grabado, demo cargada, endpoint probado, guion mejorado, backup probado, instalador revisado o candidato comercial definido. No vale solo escribir una reflexion.
