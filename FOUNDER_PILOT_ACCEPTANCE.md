# Evidentia - Aceptacion Piloto Fundador

Fecha: 2026-06-26

## Objetivo Del Piloto

Validar en 30 dias si un Evidentia Local Node convierte conocimiento disperso del cliente en memoria privada consultable con fuentes, sin vender diagnostico, decision clinica ni software medico.

## Entregables Comprometidos

1. Nodo local instalado o preparado en entorno acordado.
2. Base inicial con datos ficticios, anonimizados o autorizados por el cliente.
3. Al menos una carga guiada de conocimiento real o representativo.
4. Chat RAG con fuentes visibles sobre el contenido indexado.
5. Mapa o vista de registros para revisar conexiones entre casos, notas, archivos y aprendizaje.
6. Export knowledge bundle o pack revisable con registros, fuentes y limites.
7. Backup local creado y restauracion verificada en staging o carpeta temporal.
8. Memory report final con valor encontrado, huecos detectados y siguiente decision.

## Puerta De Entrada

No iniciar un piloto fundador si falta cualquiera de estos puntos:

- Responsable del cliente identificado.
- Datos de prueba definidos como ficticios, anonimizados o autorizados.
- Limite verbal y escrito: Evidentia no diagnostica, no prescribe y no sustituye criterio profesional.
- Entorno de instalacion acordado.
- Criterio de exito elegido antes de cargar datos.
- Plan de backup antes de manipular informacion sensible.
- Decision Pack completado desde PEDRO_PILOT_DECISION_PACK_TEMPLATE.md, con dolor comprable, 3 fuentes permitidas, pregunta real de demo, validador, precio y siguiente accion.

## Criterios De Exito

El piloto se considera valioso si cumple al menos 2 de 4 senales:

1. Recupera una decision, protocolo o aprendizaje que antes costaba encontrar.
2. Conecta al menos 3 fuentes dispersas alrededor de un caso, tema o proceso.
3. Detecta un hueco de documentacion antes de cerrar un protocolo o aprendizaje.
4. Reduce dependencia de una sola persona para explicar criterio acumulado.

## Criterios De No-Go

Parar o no convertir a continuidad si ocurre cualquiera de estos puntos:

- El cliente espera diagnostico, prescripcion o decision automatica.
- No hay datos autorizados o anonimizables para probar valor.
- Las respuestas no muestran fuentes utiles despues de la carga inicial.
- El nodo local no puede mantenerse con backup y recuperacion verificables.
- El responsable no puede dedicar feedback semanal durante el piloto.

## Preguntas De Venta Antes De Demo

1. Que conocimiento pierdes hoy entre audios, fotos, PDF, mensajes, notas o memoria humana?
2. Que decision o protocolo tarda demasiado en reconstruirse?
3. Que fuentes deberia recuperar Evidentia para que el piloto sea obviamente util?
4. Quien valida si una respuesta con fuentes es suficiente?
5. Que datos no deben entrar nunca en el piloto?
